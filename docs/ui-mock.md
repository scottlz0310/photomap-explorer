|               |                          |                     |
|               |                          | +----------------+  |

| ℹ️ ステータスバー：画像サイズ、撮影日時、通知など              |
+-------------------------------------------------------------+
📌 Component Summary | UIコンポーネント一覧
セクション	内容・仕様
📁 フォルダ選択	QTreeView + QFileSystemModel により画像フォルダを階層選択可能
🖼 サムネイル一覧	QListWidget ベースで写真を一覧表示。将来的に FlowLayout なども検討
🔍 プレビューパネル	QLabel による拡大表示。Exif 情報と一体、または切り替え可能な構成
◀ ▶ ナビゲーション	前後の画像に切り替えるボタン。キーボードとの連動も検討済
🌍 地図ビュー（Leaflet）	QWebEngineView にて HTMLベース地図を表示。GPS付き画像のピン表示対応
ℹ️ ステータスバー	QStatusBar もしくは QLabel で現在画像サイズ・Exif・通知情報を表示予定
🔧 デザインポリシー | Design Principles
直感的な情報フロー：左→右／上→下の視線誘導に沿った構成

Exif 情報の柔軟な表示制御：設定メニュー or トグルで表示ON/OFFを切替可能に

地図ビューを補助的に配置：プレビューの下部に独立表示し、没入を妨げない

レイアウト全体は QSplitter によって自由なリサイズを可能とする

キーボード操作／マウス操作の両対応を重視：操作性を高める一貫性設計

🚧 将来の UI 拡張案 | Future Enhancements
画像プレビューからの全画面表示

拡大ボタン or ダブルクリックでフルスクリーン切替

Escキーまたは戻る操作で復帰

◀ ▶ ボタンだけでなく、左右矢印キーでも画像切替

通常表示・全画面モード問わず同一操作性を提供

オーバーレイ式のExif表示やトランジション効果など

ドラッグ＆ドロップによるフォルダ選択・画像登録（UI簡略化）

- 🌟 お気に入りフォルダ機能（将来的に）
  - ユーザーが選択した画像フォルダを「お気に入り」として登録／固定表示可能に
  - 自動でピクチャフォルダ等をクロールせず、プライバシーと操作感を尊重
  - 設定はローカルファイルに永続化予定（例：config.json）

  ## 🕓 Pending Features
